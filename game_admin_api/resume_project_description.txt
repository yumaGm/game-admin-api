Game Admin API｜Python / FastAPI / Pydantic / JSON

项目描述：
基于 FastAPI 实现的游戏角色管理后端接口，通过 JSON 文件完成角色数据持久化，支持角色查询、新增、修改和删除等基础 CRUD 操作，并提供请求数据校验和异常处理功能。

主要工作：

1. 使用 GET、POST、PUT、DELETE 请求实现角色数据的查询、新增、修改和删除。
2. 使用 Pydantic BaseModel 定义角色创建和修改的数据模型，对角色名、阵营、等级、好感度和技能等字段进行基础类型校验。
3. 使用路径参数实现指定角色的查询、修改和删除功能。
4. 使用 json.load() 和 json.dump() 完成角色数据的读取、写入和持久化。
5. 添加角色时进行重名检查，角色已存在时返回 409 Conflict。
6. 查询、修改或删除不存在的角色时，通过 HTTPException 返回 404 Not Found。
7. 使用 FastAPI 自动生成 Swagger 接口文档，并通过 /docs 完成接口测试。

已实现接口：

* GET /：返回项目欢迎信息。
* GET /characters：查询全部角色。
* GET /characters/{name}：按角色名查询指定角色。
* POST /characters：添加新角色。
* PUT /characters/{name}：修改指定角色的等级和好感度。
* DELETE /characters/{name}：删除指定角色。

技术栈：
Python、FastAPI、Pydantic、RESTful API、JSON、Swagger、HTTPException、数据持久化。

项目收获：
通过该项目，我初步理解了后端 API 的基本工作方式，掌握了 GET、POST、PUT、DELETE 请求的基础使用，能够使用路径参数和请求体接收数据，并通过 Pydantic 完成基础数据校验和接口异常处理。
